✅ **完全适合您，如果：**
- 您做外贸/出口业务，有很多海外客户
- 您经常需要群发产品推广邮件
- 您觉得手动发邮件太慢、太累

### 3.1 📧 邮箱管理中心

#### 这是什么？
**这是系统的"邮局"，管理您所有的邮箱账号。**

#### 小白理解版：
```
传统方式:
您有80个邮箱账号 → 要记住80组密码 → 登录80次 → 很麻烦!

GlobalReach方式:
您配置一次(或让技术员帮忙配好) → 系统自动管理 → 您不用管!
```

#### 支持哪些邮箱？
✅ Gmail (谷歌邮箱)  
✅ Outlook (微软邮箱)  
✅ 企业邮箱 (公司自己的域名邮箱)  
✅ QQ邮箱  
✅ 163网易邮箱  

#### 🔴 初阶用法（刚上手）

**您只需要知道：**
1. 系统已经帮您管理了80个邮箱
2. 每个邮箱分配了100个客户（总共8000个）
3. 发送时系统会自动选择合适的邮箱
4. **您不需要登录任何邮箱！**

**操作步骤：**
```
Step 1: 首次使用时，提供80个邮箱的登录信息
        （或让IT人员帮您录入到系统中）
        
Step 2: 之后每次使用，直接打开系统即可
        
Step 3: 如果某个邮箱出问题，系统会通知您
```

#### 🟡 进阶用法（用熟了之后）

**您可以：**
- 查看每个邮箱的发送状态（今天发了多少封）
- 查看哪些邮箱正常、哪些被限制
- 手动调整某些邮箱的客户分配

**查看方法：**
```bash
打开文件: D:\GlobalReach\data\email-accounts\accounts.json
（这个文件是加密的，只能通过系统界面查看）
```

#### 🟢 高阶玩法（深度使用者）

**高级功能：**
- 动态负载均衡：自动把发送任务分配给最空闲的邮箱
- 智能轮换：避免某个邮箱发送过多被封号
- 健康监控：实时检测每个邮箱的"健康度"
- 应急切换：某个邮箱出问题，自动换备用邮箱

**典型场景：**
```
场景: 双11/黑五促销期间，发送量暴增3倍
解决: 系统自动启用备用邮箱池，确保发送不中断
效果: 即使在高峰期，也没有邮箱被封号
```

### 3.2 📦 产品资料投放区 (inbox)

#### 这是什么？
**这是您和系统交互的最主要入口——就像往机器里投原料！**

#### 小白理解版：
```
想象一台自动咖啡机:
├─ 您要做的是: 放入咖啡豆(产品资料)
├─ 机器自动: 磨粉→冲泡→装杯
└─ 您拿到的是: 一杯香浓的咖啡(发出的邮件)

GlobalReach就是这台"邮件咖啡机":
├─ 您要做的是: 把产品资料丢进inbox文件夹
├─ 系统自动: 解析→生成邮件→按时发送
└─ 客户收到的是: 专业的个性化推荐邮件
```

#### 🔴 初阶用法（必看！日常主要操作）

**操作步骤（每天一次即可）：**

```
Step 1: 打开文件夹
        D:\GlobalReach\workspace\inbox\
        
Step 2: 创建今天的日期文件夹
        例如: 2026-05-09\ (必须是 年-月-日 格式)
        
Step 3: 复制产品资料进去
        支持的文件类型:
        ✅ PDF文件 (产品目录、价格表)
        ✅ Excel表格 (产品清单)
        ✅ 图片文件 (产品照片)
        ✅ Word文档 (详细说明)
        
Step 4: 文件命名建议
        ✅ 好的名字: "2026年Q2新产品目录.pdf"
        ✅ 好的名字: "5月份价格表-玩具类.xlsx"

        ❌ 坏的名字: "1.jpg"
        
Step 5: 完成! 
        系统会每5分钟自动检查一次
        发现新文件后自动开始处理
```

### 3.3 ⚡ 自动发送引擎

#### 这是什么？
**这是系统的"心脏"——负责把邮件准时送到客户手中。**

#### 小白理解版：
```
传统发邮件的问题:
❌ 您在中国晚上10点发 → 美国客户凌晨收到 → 觉得被打扰
❌ 一下子发太多 → 邮箱服务商觉得您是垃圾邮件 → 封号
❌ 忘记发 → 错过最佳时机

GlobalReach的解决方案:
✅ 智能时区: 自动算出客户的当地时间，在9-11点发送
✅ 安全频率: 每3秒发1封，每个邮箱每天最多100封
✅ 全自动: 设好后不用管，到点就发
```

#### 🔴 初阶用法（您几乎不需要操作！）

**系统自动做的事：**
```
时间轴示例 (假设您在北京):

08:00  系统启动，检查今日任务
        ↓
09:00  开始发送第一批 (欧洲客户的时间是早上)
        ↓ (每3秒1封)
12:00  午间继续发送 (美洲客户的时间是深夜/凌晨，暂不发)
        ↓
14:00  下午继续 (中东/非洲客户)
        ↓
17:00  最后一批 (美洲西海岸客户刚开始上班)
        ↓
17:30  今日发送任务全部完成!
        ↓
总计: 成功发送 8,000封 (约4小时)
```

**您需要做的：**
```
✅ 确保 inbox 里有产品资料 (见3.2章节)
✅ 确保 data\clients\ 里有客户数据
✅ 其他什么都不用做!

如果想暂停发送:
→ 删除或重命名 inbox\日期文件夹 即可
```

#### 🟡 进阶用法（精细控制）

**查看发送进度：**
```
位置: D:\GlobalReach\workspace\outbox\2026-05-09\
内容: 
├── send-log.json     (详细日志: 谁收到了、什么时候发的)
├── success-list.txt  (成功发送的邮箱列表)
└── failed-list.txt   (失败的列表及原因)
```

**调整发送策略：**
```yaml
修改配置文件: D:\GlobalReach\.config\config.yaml

可调整项:
├── seconds_between_emails: 3  (改为5=更安全但更慢)
├── max_per_account_daily: 100 (改为50=更保守)
└── sending_hours_start: "09:00" (改为10:00=晚点开始)
```

**处理失败邮件：**
```
系统会自动重试1次 (2小时后)
如果还是失败:
→ 在 failed-list.txt 中查看原因
→ 常见原因及解决:
   ├─ "邮箱不存在" → 更新客户数据库
   ├─ "被拒收" → 该客户可能退订了
   └─ "临时错误" → 明天会再试
```

**A/B测试功能：**

```
测试不同主题行的效果:
```

### 3.4 🔍 回复收集中心

#### 这是什么？
**这是系统的"耳朵"——帮您监听所有客户的反馈。**

#### 小白理解版：
```
传统方式的痛点:
❌ 80个邮箱, 要一个个登录看
❌ 忙起来忘了检查, 错过重要回复
❌ 客户说"urgent"(紧急), 但您隔天才看到

GlobalReach的方式:
✅ 每2小时自动检查所有邮箱
✅ 有紧急邮件立刻弹窗提醒您
✅ 所有回复自动整理到一个地方
✅ AI帮您分析客户想干什么
```

#### 🔴 初阶用法（最重要！必须掌握）

**系统自动做的事：**
```
时间表 (每天):
08:00  第一次检查 (看看昨晚有没有新回复)
10:00  第二次检查 (上午的回复)
12:00  第三次检查 (午间的)
14:00  第四次检查
16:00  第五次检查
18:00  第六次检查 (下午最后一批)
20:00  第七次检查 (傍晚的)
22:00  第八次检查 (夜间的)
```

**您需要做的：**
```
Step 1: 查看"草稿区"
        D:\GlobalReach\workspace\drafts\2026-05-09\
        
Step 2: 看到AI已经帮您写好了回复草稿
        文件名示例:
        ├── reply-to-john@abc.com_询价RC赛车.md
        ├── reply-to-maria@xyz.com_下单意向.md
        └── URGENT-reply-to-tom@bigbuyer.com_紧急订单.md
        
Step 3: 点开查看内容
        AI会标注:
        ├── 【客户原意】: 他想要什么
        ├── 【建议回复】: AI帮您写的草稿
        └── 【待您决定】: 哪些地方需要您确认
        
Step 4: 三种选择
        ✅ 直接确认 → 点击"发送"
        ✏️ 修改后再发 → 编辑文字 → 点击"发送"
        ⏸️ 暂时不处理 → 标记"稍后处理"
```

**紧急邮件识别：**
```
以下关键词触发"红色警报", 系统会立刻通知您:
├── order / 订单
├── urgent / 紧急
├── MOQ (最小起订量)
├── price / 价格
├── sample / 样品
├── payment / 付款
└── shipment / 发货

收到这类邮件:
→ 弹窗提醒 + 手机通知(可选)
→ 自动排在草稿区最前面
→ 标注为 🔴 高优先级
```

#### 🟡 进阶用法（高效处理）

**批量处理模式：**
```
当积压了很多回复时:

1. 打开 drafts 文件夹
2. 按"优先级"排序:
   🔴 紧急 (order/urgent) → 先处理
   🟡 意向 (询价/样品) → 其次处理
   🟢 一般 (谢谢/再见) → 可批量确认
3. 批量操作:
   → "全部确认发送" (针对一般性回复)
   → "逐个审核" (针对重要客户)
```

**客户意图自动分类：**
```
AI会把回复分成几类:

📈 下单意向 (占15%)
   → 客户说:"I want to place an order"
   → 优先级: 最高
   → 建议: 立即人工介入, 谈具体细节

💰 询价询问 (占40%)
```

A: 我们做了很多防垃圾邮件设计:
- 每封邮件都是个性化撰写 (不是千篇一律)
- 发送频率控制 (每3秒1封, 很克制)
- 单账号日限100封 (远低于警戒线)
- 内容质量高 (AI写的比很多人写得还好)

只要您不发违法/欺诈内容, 基本不会被判定为垃圾邮件。

**Q4: 邮箱会不会被封?**
A: 极低概率。因为:
- 我们用的是正规企业邮箱为主
- 发送行为模拟真人 (不会半夜狂发)
- 有自动保护机制 (检测到异常立刻停发)

万一真的被限制, 系统会通知您, 通常解封很简单。

### ❓ 进阶问题

**Q5: AI写的邮件能看出来是机器写的吗?**
A: 前2周可能稍微有点"模板感"。但经过**蒸馏学习**后, AI会逐渐模仿您的风格, 1个月后基本看不出区别。

**Q6: 可以同时用多个产品资料吗?**
A: 可以! 只要都放进当天的inbox文件夹就行。系统会自动合并成一个推广主题。

**Q7: 客户回复"取消订阅"怎么办?**
A: 系统会自动标记该客户为"退订"状态, 不会再给他发营销邮件。(符合国际反垃圾邮件法规)

**Q8: 数据安全吗? 我的客户名单会泄露吗?**
A: **绝对不会!** 
- 所有数据存储在**您自己的电脑** (本地Windows)
- 采用**军用级加密** (AES-256)
- **不上传到任何云服务器**
- 即使电脑被盗, 没有密码也解不开
